VLSI Agent v3 — Full C++ System

Implementation Plan · C++23 (C++17 compatible) · 13 libraries · 57+ classes · pybind11 · Boost.Beast

1. Overview

This document specifies the complete C++ implementation for the VLSI Agent system. It covers the routing engine, six MCP servers, WebSocket transport, Python bindings, and the build infrastructure needed to produce production-grade chip routing at 1 M-net scale.

City Analogy — Think of a routed chip as a city's road network. GlobalPlanner is the city planner sketching highway corridors. NegotiatedRoutingLoop is traffic management negotiating conflicting routes. ThreadManager is the city's transport authority — it alone owns all the vehicles. MCP Servers are the city's service departments (database, windows, placement…), each reachable by a clean JSON-RPC API.

2. System Architecture

The system is split into two layers: a Python LangGraph orchestrator (the AI brain) and the C++ execution layer (pure compute, no LLM dependency). Communication crosses the boundary via JSON-RPC 2.0 over HTTP/WebSocket for out-of-process servers, or direct pybind11 calls for in-process use.

RoutingGridGraph — core/grid_graph.hpp

Builder Flyweight std::mdspan std::atomic CAS

The 3D chip routing grid. Edges represent legal tracks and vias. Ownership is managed via atomic CAS — no mutex on the hot routing path.

class RoutingGridGraph {
public:
    class Builder {
    public:
        Builder& set_dimensions(int rows, int cols, int layers);
        Builder& set_edge_weights(std::shared_ptr<EdgeWeightTable>);
        [[nodiscard]] RoutingGridGraph build();
    };

    // Try to claim an edge for net_id. Returns false if already owned.
    [[nodiscard]] bool try_claim_edge(EdgeRef e, NetId net_id) noexcept;
    void release_edge(EdgeRef e, NetId net_id) noexcept;

    // Freeze a net's edges (ECO: prevents rip-up of frozen nets)
    void freeze_net(NetId net_id) noexcept;
    [[nodiscard]] bool is_frozen(NetId net_id) const noexcept;

    [[nodiscard]] std::optional<EdgeRef> edge(GridPoint from, GridPoint to) const noexcept;
};

Threading: edge ownership via std::atomic<uint32_t> CAS. Freeze mask written once before threads start — all threads read-only thereafter. Flyweight: EdgeWeightTable is shared across all graph instances (same process node).

NegotiatedRoutingLoop — routing/negotiated_routing_loop.hpp

State Machine Observer std::variant state ranges

PathFinder outer loop. Iterates: route all nets → collect conflicts → update history costs → check convergence → ILP fallback if oscillating. ConvergenceMonitor is registered as an observer.

Traffic Cop Analogy (Negotiation) — Imagine 10,000 drivers (nets) who all want to take the exact same highway at rush hour. If they drive blindly, they crash. The Negotiated Routing Loop acts as a Master Traffic Cop managing a simulation. It lets everyone drive, sees exactly where the crashes happen, and then rips up the crashed cars, erecting temporary toll booths (Cost Penalties) in those exact lanes. On the next pass, drivers naturally route themselves around the temporary toll booths. This loop repeats until 0 crashes occur. 
// States as variant tags
struct RoutingState{};   struct CollectingState{};
struct UpdatingState{};  struct CheckingState{};
struct IlpFallbackState{}; struct ConvergedState{};
using NrlState = std::variant<RoutingState, CollectingState,
                               UpdatingState, CheckingState,
                               IlpFallbackState, ConvergedState>;

class NegotiatedRoutingLoop {
public:
    // Observer registration
    void register_observer(ConvergenceMonitor&);
    void register_observer(AdaptivePenaltyController&);

    [[nodiscard]] std::expected<RouteResult, RoutingError>
        run(RoutingGridGraph&, const CorridorAssignment&,
            HistoryCostUpdater&, ThreadManager&);
};

HistoryCostUpdater — routing/history_cost_updater.hpp

Observer Chain of Responsibility std::atomic<double>

Dual-signal cost updater. Congestion channel (from NRL conflicts) and DRC channel (from DrcPenaltyModel masks) are merged with independent weights W_cong and W_drc.

Toll Booth Analogy (History Cost) — If the Negotiated Routing Loop is the Traffic Cop, the History Cost Updater is the Toll Booth Operator. Every time a crash happens on a track, the Operator permanently increases the toll price for that specific track for all future iterations. The more a track is fought over, the more expensive it becomes for the A* router to pick it. 
class HistoryCostUpdater {
public:
    // Called by NRL after each iteration
    void on_conflict(const ConflictEvent&) noexcept;

    // Called by DrcPenaltyModel when a rule mask fires
    void on_drc_violation(EdgeRef, float severity) noexcept;

    // Returns merged cost for DetailedGridRouter weight W3
    [[nodiscard]] float merged_cost(EdgeRef) const noexcept;

    void set_weights(float w_cong, float w_drc) noexcept;
};
// Storage: std::atomic<double> per edge (lock-free on x86/arm64)
// W_cong and W_drc independently tunable by AdaptivePenaltyController

DetailedGridRouter — routing/detailed_grid_router.hpp

Template Method Object Pool Concepts [[likely]]

A* maze router. The skeleton is fixed (open/closed sets, relaxation, path reconstruction). Three pluggable Concept-constrained weight sources, plus a fourth EM/IR adjustment channel.

Whiteboard Analogy (Memory Pooling) — Imagine organizing a library by handing out a brand-new notebook (OS memory allocation) to every single patron (an A* Node) who enters, and throwing it away when they leave. The library would crash from administrative overhead! Instead, we use boost::object_pool. It acts like a stack of fast, reusable whiteboards. When an A* path expands, it grabs a whiteboard, evaluates its costs, and upon path closure, wipes it clean and puts it back. This completely eliminates OS allocation overhead, allowing the A* engine to expand millions of nodes per second! 
template <typename W>
concept WeightSource = requires(W w, EdgeRef e) {
    { w.cost(e) } -> std::convertible_to<float>;
};

class DetailedGridRouter {
public:
    template <WeightSource W1, WeightSource W2, WeightSource W3, WeightSource W4>
    [[nodiscard]] std::expected<RoutedPath, RouterError>
        route(GridPoint src, GridPoint dst, NetId net,
              RoutingGridGraph&,
              W1 base_cost,       // track base weight
              W2 cong_history,    // HistoryCostUpdater output
              W3 drc_avoidance,   // DrcPenaltyModel mask
              W4 em_ir_adj);      // ElectricalConstraintEngine pre-compute

    // Heuristic, edge weight, expansion legality — Template Method hooks
    virtual float compute_heuristic(GridPoint cur, GridPoint dst) const;
    virtual float edge_weight(EdgeRef) const;
    virtual bool  is_legal_expansion(EdgeRef, NetId) const;
};
// [[likely]] on visited-set cache hit in the hot inner loop

ConvergenceMonitor — convergence/convergence_monitor.hpp

Observer State Machine std::optional

Tracks conflict counts across NRL iterations. Detects oscillation (conflict count cycles without progress). Triggers ILP fallback on the oscillating sub-region.

// States
struct ConvergingState{};
struct OscillatingState{};
struct IlpTriggeredState{};
using CmState = std::variant<ConvergingState, OscillatingState, IlpTriggeredState>;

class ConvergenceMonitor {
public:
    // Called by NRL after each iteration (Observer interface)
    void on_iteration(int conflict_count, int pass_number);

    [[nodiscard]] bool is_converged() const noexcept;

    // Non-empty when oscillation detected → IlpSolver should isolate this region

6. C++ Features Guide

[[nodiscard]]

The Unopened Mail Analogy — Imagine a postman hands you a highly important certified letter, and you immediately throw it into the trash without opening it. The postman would trigger an alarm! `[[nodiscard]]` mathematically forces the C++ compiler to throw a warning if an EDA function generates a route or retrieves an object ID, and the programmer forgets to capture the result in a variable to use it.

std::expected<T, Error> (C++23)

The Package Delivery Analogy — When you accept a package, you either open the box to find your actual item (the value), OR you find an apology note stating why it couldn't be delivered (the error). You cannot have both, and you cannot have neither. This forces the programmer to formally handle failure states (like DRC routing failures) rather than returning a naked `bool` and hoping the caller checks it.

[[likely]] / [[unlikely]] (C++20)

The Train Switch Analogy — If 99% of trains go straight and 1% turn left, operators will lock the physical tracks to "straight" so trains never even have to slow down. `[[likely]]` instructs the CPU hardware to pre-load the "straight path" instructions into the L1 CPU cache. In the hot path of the A* router, we label the cache-hits as `[[likely]]` to massively increase iteration speed.

std::span<T> (C++20)

The Glass Window Analogy — If your friend wants to read one specific paragraph from your 1,000-page book, photocopying the whole book is incredibly wasteful. Instead, you just place a small glass window isolating exactly that paragraph. `std::span` is a zero-copy pointer window into an existing array in memory (like a subset of `NetIds`), avoiding massive memory duplication!

std::barrier (C++20)

The School Field Trip Analogy — Imagine a teacher taking 30 students to a museum. They all explore at different speeds, but the teacher stands at the exit. The bus cannot leave until all 30 students have arrived at the exit. std::barrier forces different thread regions to wait at a checkpoint line until every single thread finishes its work, ensuring safely synchronized progression!

std::shared_mutex (C++17)

The Museum Exhibit Analogy — 100 tourists can peacefully look at a painting in a room at the exact same time (Shared Read). However, if an engineer needs to swap the painting to a new physical frame, they must kick everyone completely out of the room until they are done (Exclusive Write). std::shared_mutex allows millions of fast simultaneous reads without blocking, only locking when a write occurs!

std::variant (C++17)

The Swiss Army Knife Analogy — A std::variant is like a physical Swiss Army Knife. The object can pivot to deploy a blade, a corkscrew, or a screwdriver. But physically, it can only act as ONE of those exact tools at any given second. It prevents messy type errors by strictly enforcing that the object contains exactly one explicitly defined state.

boost::container::flat_map

The Rolodex Analogy — Standard maps store data scattered randomly across a room (Heap Fragmentation). A flat_map forces all identical data to be rigidly printed onto a tightly packed desktop Rolodex. Because memory is contiguous, the CPU cache can read hundreds of records simultaneously without jumping across RAM, making data retrieval phenomenally fast!

Universal version-check idiom used throughout every header:

#else
    // C++17: fallback implementations
#endif

8. Threading Guide

The one rule: Only threading/thread_manager.hpp (and its .inl and .cpp) may #include <thread>, <barrier>, <latch>, or <mutex>. Every other class receives a ThreadManager& reference. Violations fail code review.

Worker Pool Flow

main thread
  │  ThreadManager tm(16);  // 16 jthread workers
  │
  ├─► SpatialPartitioner.partition_and_route(graph, tm, nrl)
  │     │  tm.submit([&]{ region_0_route(); })  // jthread worker 0
  │     │  tm.submit([&]{ region_1_route(); })  // jthread worker 1
  │     │  ...
  │     │  barrier = tm.make_barrier(N_regions)
  │     │
  │     │  [all region workers run NRL independently]
  │     │
  │     ▼  [all workers arrive at barrier]
  │
  └─► CrossRegionMediator.mediate(graph, barrier, cross_nets, router)
        [runs serially on main thread; no races on ghost cells]
        barrier.arrive_and_wait()  // releases region workers for next pass

Edge Ownership Protocol (RoutingGridGraph)

// Claim an edge with CAS — lock-free, safe from N threads simultaneously
bool claimed = graph.try_claim_edge(edge_ref, my_net_id);
if (!claimed) { /* handle conflict, update history cost */ }

// Release when ripping up
graph.release_edge(edge_ref, my_net_id);

// Frozen edges (ECO) — written once before threads start
graph.freeze_net(power_net_id);
// All threads: is_frozen() is read-only; no synchronization needed

Ghost Cell Protocol

When SpatialPartitioner divides the grid into regions, boundary cells are marked as ghost cells — read-only from the perspective of the region worker. The CrossRegionMediator is the only code that writes to ghost cells, and it does so after all region workers have reached the barrier.

Region 0 boundary  │  Ghost cells (read-only for R0 and R1)  │  Region 1 boundary
────────────────────┼─────────────────────────────────────────┼────────────────────
R0 worker reads     │  [CrossRegionMediator writes here]       │  R1 worker reads
no writes           │  (only after barrier sync)              │  no writes

std::string dispatch_tool(std::string_view name,
                           const nlohmann::json& params) override {
    std::promise<nlohmann::json> promise;
    auto future = promise.get_future();

    // Post to Qt main thread via QueuedConnection
    QMetaObject::invokeMethod(qt_widget_, [&, p = std::move(promise)]() mutable {
        p.set_value(execute_on_qt_thread(name, params));
    }, Qt::QueuedConnection);

    return future.get().dump(); // blocks I/O thread until Qt responds
}
